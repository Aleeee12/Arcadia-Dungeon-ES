📋 Résumé Exécutif

Arcadia Dungeon est un mod NeoForge server-side conçu comme couche d'endgame RPG pour les serveurs Minecraft fortement moddés. Là où le contenu existant est rapidement maîtrisé par les joueurs end-game, Arcadia offre une progression verticale continue via des donjons entièrement configurables par JSON.

Chaque élément est pilotable sans recompilation : boss multi-phases avec seuils de PV, invocations, multiplicateurs, dodge, barres de vie, vagues ordonnées, récompenses cross-mods, cooldowns par joueur, leaderboard hebdomadaire, confinement de zone, commandes admin et rechargement à chaud. Le mod se positionne comme hub d'endgame cross-mods — les récompenses peuvent être n'importe quel item de n'importe quel mod installé sur le serveur.

Réaction cible admin : "Ils ont pensé à tout" — chaque cas d'usage gameplay a une option de configuration dédiée.

✨ Ce qui le rend unique

4 piliers de différenciation par rapport aux alternatives existantes (Dungeons+, Better Dungeons, etc.)

⚙️

Configuration totale par JSON

Boss, phases, vagues, récompenses, spawn points, messages, cooldowns, aire de confinement — tout est pilotable sans toucher au code Java ni redémarrer le serveur.

🔗

Hub d'endgame cross-mods

Les récompenses peuvent être n'importe quel item Minecraft ou mod tiers. Arcadia devient le point de convergence des progressions sur un serveur à 400+ mods.

🤖

Administration naturelle via IA

Un skill Claude Code conversationnel permet à l'admin de créer et modifier des donjons en langage naturel — sans ouvrir un seul fichier JSON manuellement.

🏆

Compétition & rétention

Leaderboard hebdomadaire par donjon, cooldowns par joueur, speedrun reproductible. Les joueurs reviennent chaque semaine. Les donjons sont discutés sur Discord.

⚡ Exigences Non-Fonctionnelles Critiques

18 NFR répartis en 5 catégories — les contraintes qui déterminent les décisions architecturales.

⚡ Performance

  • Tick handler < 1ms par tick (NFR1)
  • Pattern nextXCheckAt obligatoire (NFR2)
  • Event handlers de combat : O(1) (NFR3)
  • Démarrage donjon < 500ms (NFR4)
  • 3 donjons actifs ≥ 18 TPS (NFR5)

🛡️ Fiabilité

  • Crash joueur sans bloquer le donjon (NFR6)
  • Arrêt serveur : sauvegarde cooldowns/leaderboard (NFR7)
  • JSON invalide n'empêche pas les autres (NFR8)
  • Hot reload sans interruption (NFR9)
  • Zéro exception non catchée (NFR10)

🔗 Intégration

  • Aucune dépendance obligatoire (NFR11)
  • Mod absent → log WARN, pas crash (NFR12)
  • Compatible mods d'attributs (NFR13)
  • JSON rétrocompatible (NFR14)
  • Sections Spark visibles (NFR15)

🔒 Sécurité

  • Commandes → permission op niveau 2 (NFR16)
  • Persistance : UUID + timestamps uniquement (NFR17)
  • Skill IA : validation avant écriture (NFR18)

🏗️ Architecture Technique

Pattern Singleton sur tous les managers, double event bus NeoForge, frontières de packages strictes.

📦 Packages principaux

  • util/ — ModCompat, SparkUtil, MessageUtil
  • config/ — POJOs JSON, ConfigManager
  • dungeon/ — DungeonManager, DungeonInstance
  • boss/ — BossManager, BossInstance
  • event/ — DungeonEventHandler
  • command/ — ArcadiaCommands, AdminCommands

🔄 Patterns clés

  • Singletons — tous les managers
  • scheduleRemoval() — anti-ConcurrentModification
  • nextXCheckAt — throttling systématique
  • "arcadia_managed" — tag entités gérées
  • ArcadiaCombat NBT — sous-tag combat
  • try/catch global — event handlers

💾 Persistance

  • Configs → arcadia/dungeon/dungeons/*.json
  • Joueurs → arcadia/dungeon/players/{uuid}.json
  • Leaderboard → JSON, reset hebdomadaire
  • Sérialisation → Gson setPrettyPrinting
  • Hot reload → instances actives non affectées

⚠️ Anti-patterns interdits

  • new ResourceLocation() → ResourceLocation.parse()
  • player.hasPermissions() → PermissionAPI
  • activeInstances.remove() → scheduleRemoval()
  • player.sendSystemMessage() → MessageUtil
  • ModList.get().isLoaded() → ModCompat.HAS_*
  • System.out.println() → LOGGER

🚀 Roadmap — 3 Phases

Approche MVP "Experience" — qualité production avant publication CurseForge, zéro scope creep.

Phase 1 · En cours
MVP — Production Ready
  • Config JSON complète (vagues, boss multi-phases)
  • Combat tuning configurable
  • Stabilité production (anti-parasite, containment)
  • Leaderboard hebdomadaire
  • Recrutement avec annonce serveur
  • Commandes admin complètes
  • Schéma JSON + exemple annoté
  • Skill Claude Code admin (privé)
Phase 2 · Post-CurseForge
Growth
  • Progression joueur (niveaux, points, rang RPG)
  • Conditions avancées (prérequis cross-mods)
  • Nouvelles mécaniques (pièges, objectifs custom)
  • Interface stats visible in-game
Phase 3 · Vision
Couche RPG Complète
  • Client-side : items custom, textures, UI dédiée
  • Intégration profonde cross-mods
  • Classes, talents, équipement Arcadia
  • Synchronisation multi-serveurs (à explorer)
